**File 1: `pipes/WDL/workflows/parse_kraken2_reads.wdl`** (per REQ-08)

Create this file with the exact content below, modeled on `classify_reads_by_contig.wdl`:

```wdl
version 1.0

import "../tasks/tasks_metagenomics.wdl" as metagenomics

workflow parse_kraken2_reads {
    meta {
        description: "Parse Kraken2 per-read output and annotate each read with NCBI taxonomy (name, kingdom, rank) using a pre-built DuckDB taxonomy database."
        author: "Broad Viral Genomics"
        email:  "viral-ngs@broadinstitute.org"
        allowNestedInputs: true
    }

    input {
        File    kraken2_reads_output
        File    taxonomy_db
        String  sample_id = sub(basename(kraken2_reads_output), "\\.kraken2\\.reads\\.txt(\\.gz)?$", "")
        Boolean resolve_strains = false
    }

    call metagenomics.parse_kraken2_reads {
        input:
            kraken2_reads_output = kraken2_reads_output,
            taxonomy_db          = taxonomy_db,
            sample_id            = sample_id,
            resolve_strains      = resolve_strains
    }

    output {
        File read_taxonomy = parse_kraken2_reads.read_taxonomy
    }
}
```

- Do NOT include `docker` as a workflow input (per Pitfall 6, matches neighbor pattern)
- `allowNestedInputs: true` in meta block (per REQ-08)
- Input defaults match the task defaults exactly (per D-05)
- 4 inputs passed through to task call (no docker pass-through)
- Output name `read_taxonomy` matches the task output name
